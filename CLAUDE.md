# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Nix-darwin + home-manager dotfiles for macOS (Apple Silicon). Declarative system configuration using Nix Flakes.

## Key Commands

```bash
rebuild                    # Apply all changes (alias for darwin-rebuild switch --flake ~/.dotfiles)
nix-collect-garbage -d     # Clean old generations
```

After modifying:

- `.nix` files: run `rebuild`
- `config/emacs/init.el`: restart Emacs
- `config/ghostty/config`: restart terminal
- Fish config: reload shell (`exec fish`)

```
hosts/zyrbbook/
  ├── default.nix        # Darwin system config (imports darwin modules)
  └── home.nix           # Home-manager config (imports home modules)

  ├── darwin/
  │   ├── homebrew.nix   # Brew packages (casks, formulae)
  │   └── system.nix     # macOS defaults (Dock, Finder, keyboard)
  └── home/
      ├── shell/         # Fish + Starship
      ├── terminal/      # Ghostty + tmux
      ├── editors/       # Emacs (terminal-only, straight.el)
      ├── tools/         # ripgrep, fd, yazi, htop

```

## Patterns

**Module Organization**: One `.nix` file per tool. Host-specific overrides in `hosts/`.

**File Linking**: Non-Nix configs linked via `home.file`:

```nix
home.file.".emacs.d/init.el".source = ../../../config/emacs/init.el;
```

**Adding Packages**:

- GUI apps: `modules/darwin/homebrew.nix` → `casks`
- CLI tools: `modules/darwin/homebrew.nix` → `brews` or add to relevant home module
- Nix packages: add to relevant module's `home.packages`

**Adding a New Host**:

1. Create `hosts/<hostname>/default.nix` and `home.nix`
2. Add `darwinConfigurations."<hostname>"` to `flake.nix`
3. Run `darwin-rebuild switch --flake .#<hostname>`

**Theme System**: Catppuccin Mocha palette defined in `modules/home/theme/catppuccin.nix`, imported by shell/terminal tools for consistency.

## Gotchas

**tmux comma escaping**: Commas inside `#[...]` style attributes (e.g. `#[bg=red,fg=white]`) must be escaped as `#,` when nested inside tmux conditionals (`#{?...}`). Unescaped commas get parsed as conditional branch separators, breaking the format string. This applies to `window-status-format` and any other format using conditionals. The Nix source in `tmux.nix` uses `#,` but when editing `~/.tmux.conf` directly, you must escape manually.

**Deployed vs source configs**: `~/.tmux.conf` is generated by home-manager from `modules/home/terminal/tmux.nix`. Direct edits to `~/.tmux.conf` are for immediate testing only — they get overwritten on `rebuild`. Always make the canonical change in the Nix source.

**Emacs auto-pairing**: Uses built-in `electric-pair-mode`, not smartparens. Smartparens breaks skip-closing-delimiter behavior in tree-sitter modes (typing `{}` produces `{}}` because smartparens fails to detect the existing closing delimiter). Don't re-add smartparens unless structural editing (slurp/barf) is actually needed.

**Emacs terminal rendering**: Running `emacs -nw` means overlays, popups, and face changes cause visible flicker. Key rules: (1) never auto-display `eldoc-doc-buffer` — it opens windows unexpectedly; use echo area instead, (2) disable `:documentHighlightProvider` in eglot — symbol-at-point highlighting causes constant redraws, (3) disable `:inlayHintProvider` — inlay hint overlays bleed into buffer text in terminal, (4) disable `:semanticTokensProvider` — causes two-phase color flash (tree-sitter colors first, then semantic tokens override ~500ms later), (5) disable `:documentOnTypeFormattingProvider` — reformats code mid-keystroke, eats characters, (6) don't use `jit-lock-defer-time` — deferred fontification causes visible unstyled-then-recolored flashes, (7) keep `corfu-auto-delay` >= 0.3 and `corfu-auto-prefix` >= 3 to avoid popup thrash, (8) copilot.el ghost text overlays may eat keystrokes in terminal mode — if issues recur, check `copilot-completion-map` bindings.

**Ghostty Cmd key bindings**: Cmd keys are sent as `\x1b[N;13~` escape sequences (where N is ASCII code, 13 is the custom modifier). Both Ghostty config and Emacs `input-decode-map` must match. Currently mapped: `Cmd+.` → `eglot-code-actions`, `Cmd+Space` → treemacs, `Cmd+c` → copy.

**Claudemacs uses flymake, not flycheck**: The `claudemacs--get-flycheck-errors-on-line` and `claudemacs--format-flycheck-errors` functions are overridden in our config to use `flymake-diagnostics` instead of flycheck. Don't install flycheck — it would conflict with flymake.

## Key Configurations

- **Shell**: Fish + Starship, auto-starts tmux "main" session
- **Terminal**: Ghostty (80% opacity, VS Code Dark+ colors)
- **tmux**: Prefix `Ctrl+\`, sessions persist via continuum
- **Emacs**: Terminal-only (`emacs -nw`), 24-bit color, LSP via eglot, Claude via claudemacs
- **Git**: delta pager, rerere enabled, editor is `emacs -nw`
